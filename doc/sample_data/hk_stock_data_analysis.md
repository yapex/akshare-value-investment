- **列数**: 36列
- **结构**: 宽表结构

### 标准结构特点
1. **宽表结构**: 每行代表一个报告期，每列代表一个财务指标
2. **多期数据**: 包含2016年到2024年的9个财年数据
3. **计算指标**: 包含ROE、ROA、每股收益等各种财务比率
4. **增长率指标**: 包含同比增长率（_YOY）和环比增长率（_QOQ）

### 关键字段
| 字段名 | 描述 | 示例 |

| STD_ITEM_NAME | 标准项目名称 | 现金及现金等价物 |

### 窄表结构特点
1. **STD_ITEM_NAME字段**: 存储具体的财务项目名称
2. **AMOUNT字段**: 存储对应的数值
3. **多期数据**: 同一个STD_ITEM_NAME会出现在多个REPORT_DATE中
4. **分类清晰**: 通过STD_ITEM_CODE可以识别项目类型

## 🔍 窄表vs标准结构对比

### 窄表结构（财务三表）
**优势**:
- 数据完整性高，包含24年历史数据
- 项目分类详细，覆盖财务报表所有项目
- 灵活性强，可以轻松添加新的财务项目
- 适合深度财务分析

**劣势**:
- 数据量大（1069行 vs 9行）
- 需要数据筛选和转换
- 查询特定项目需要额外处理

### 标准结构（财务指标）
**优势**:
- 查询简单，直接使用列名
- 数据结构清晰
- 适合标准化的财务分析
- 包含重要的财务比率

**劣势**:
- 历史数据相对较少（9年）
- 灵活性差，添加新指标需要修改结构
- 项目覆盖有限

## 🎯 系统设计启示

### 1. 混合查询架构
```yaml
# 港股市场配置
hk_stock:
  indicators:
    structure: "wide"  # 宽表结构
    fields: ["BASIC_EPS", "ROE_AVG", "DEBT_ASSET_RATIO"]

  statements:
    structure: "narrow"  # 窄表结构
    key_fields: ["总资产", "总负债", "股东权益", "现金及现金等价物"]
```

### 2. 字段映射策略
```yaml
# 港股标准字段（财务指标）
"ROE_AVG":
  name: "净资产收益率"
  keywords: ["ROE", "净资产收益率", "股东回报率"]
  market: "hk_stock"
  data_type: "indicators"

# 港股窄表字段（财务三表）
"总资产":
  name: "总资产"
  keywords: ["总资产", "资产总额", "Total Assets"]
  market: "hk_stock"
  data_type: "statements"
  api_field: "STD_ITEM_NAME"
  filter_value: "总资产"
  value_field: "AMOUNT"
  field_type: "narrow"
```

### 3. 查询优化建议
1. **指标查询**: 优先使用宽表结构，查询效率高
2. **详细分析**: 使用窄表结构，支持多维度分析
3. **缓存策略**: 对窄表数据进行适当缓存，提高查询速度
4. **数据更新**: 财务三表数据更新频率较低，可以长时间缓存

## 📋 实现优先级

### 高优先级（核心字段）
